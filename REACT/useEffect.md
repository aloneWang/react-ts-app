## 释义
该hook 在组件渲染之后执行一些副作用的操作，
例如：改变dom，添加订阅，记录日志等
```
useEffect(()=>{
  // do sth
  return () => {
    // 组件卸载时的执行的操作
  }
})
```
`注意`： React 会 对 state 的变化会做优化，若更新过后的 state 与旧值一样，将跳过渲染（effect， 更新 state 的操作都会停止）
为了防止内存泄漏，组件卸载会在 返回函数中清除创建的副作用
#### effect 的条件执行
默认情况下， effect 会在每次渲染组件后执行，有时候需要某些指定的依赖变化时才执行，就可以在第二个参数传入相关的依赖数组，只有当数组的值变化时才会重新创建执行,
只传入一个空数组`[]` 时，只会运行一次
```
useEffect( () => {

},[props.source])
```
