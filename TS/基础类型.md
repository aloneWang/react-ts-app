```

```

## 数组
ts 像js 一样操作数组元素，有两种方式定义数组，第一种：

```
let list：number[] = [1,23,2]
```
第二种是使用泛型， `Array<元素类型>`

```
let list: Array<number> = [1,23,1]

```
## 元祖 Tuple
表示一个可以是不同类型的元素的数组
example:
```
let x: [string, number]
x = ['hellow', 10] // ok
x = [ 10, 'hellow'] // error
```
当访问一个越界的元素，会使用联合类型
```
x[3] = '1212' // 可以是 number || string
x[6] = true // error , 不存在 boolean 类型
```

## 枚举
```
enum Color { red, gren, blue}
let c: Color = Color.red

```
color 对象是可以双向映射的， 默认情况下是 从0开始，转义成js
```
var Color;
(function (Color) {
    Color[Color["red"] = 0] = "red";
    Color[Color["gren"] = 1] = "gren";
    Color[Color["black"] = 2] = "black";
})(Color || (Color = {}));
var c = Color.red;

```
## Any
是在不确定当前类型的情况下，指定为一个类型
```
let notSure: any = 4
notSure = '1212' // ok
```
## Void
void 类型与 any 类型恰恰相反 ，表示没有任何类型，当函数没有返回值，可以为 void
```
function warn(): void {
  console.log(11)
}
```
## Null 和 Undefined
TypeScript里，undefined和null两者各自有自己的类型分别叫做undefined和null。 和 void相似，它们的本身的类型用处不是很大：
```
let u: undefined = undefined;
let n: null = null;
```
`note`: 
默认情况下 `null` 和 `undefided` 是所有类型的子类型， 就是可以把 `null` 和 `undefined` 赋值 number类型的变量

## never 
表示那些永远不存在值的类型， 是任何类型的子类型， 也可以赋值给任何类型

## Object
object表示非原始类型，也就是除number，string，boolean，symbol，null或undefined之外的类型

## 类型断言
表示，你知道它的类型， “相信我，我知道自己在干什么”

有两种语法：
+ 尖括号
 ```
  let someValue: any = '1212'
  let length: number = (<string>someValue).length

 ```
+ as 
```
  let someValue: any = '1212'
  let length: number = (someValue as string).length

```
`注意`： 两种都是等价的，在使用jsx ,只能用as语法
